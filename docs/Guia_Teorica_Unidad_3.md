# Capítulo 3. Guía teórica de resolución de problemas estructurados y persistencia básica

Este capítulo integra lo aprendido para resolver problemas más estructurados mediante funciones, organización modular y almacenamiento elemental de información. La competencia principal es construir soluciones reutilizables y mantener la coherencia del programa cuando el problema deja de ser pequeño.

## Competencias del capítulo

- Descomponer problemas en funciones con una responsabilidad clara.
- Diseñar soluciones modulares con parámetros y retorno de valores.
- Integrar menús, colecciones y funciones en programas más completos.
- Guardar y recuperar información básica usando archivos de texto.

## Idea central

Una solución competente no solo resuelve el problema; también se puede entender, reutilizar, mantener y extender.

## Ruta de lectura del capítulo

1. Lea esta guía para reconocer la competencia integradora del capítulo.
2. Estudie cómo una función encapsula una parte del problema.
3. Observe cómo varias funciones coordinadas forman una solución completa.
4. Analice por qué la persistencia básica permite continuidad de la información.
5. Use la evaluación final como demostración de integración de competencias.

## Preguntas guía

- ¿Qué gana un programa cuando se divide en funciones?
- ¿Cómo se conectan la modularización y la claridad del algoritmo?
- ¿Qué decisiones mínimas se necesitan para persistir datos con seguridad básica?

## Evidencia esperada

Al terminar este capítulo, el estudiante debe poder construir programas modulares que integren funciones, organización de datos y almacenamiento básico de información.
