# Capítulo 1. Guía teórica de resolución de problemas básicos

Este capítulo introduce la base del pensamiento computacional aplicado a problemas simples. No se organiza como un manual tradicional de definiciones aisladas, sino como una ruta de competencias: comprender situaciones, identificar datos, decidir procesos y construir soluciones verificables.

## Competencias del capítulo

- Reconocer un problema computacional sencillo y expresarlo con claridad.
- Diferenciar entradas, procesos y salidas en una solución elemental.
- Representar soluciones con algoritmos simples antes de programarlas.
- Traducir algoritmos básicos a Python usando variables, tipos de datos y decisiones.

## Idea central

Programar no comienza escribiendo código. Comienza comprendiendo qué se necesita resolver, qué información se recibe, qué transformación se realiza y cómo se valida el resultado.

## Ruta de lectura del capítulo

1. Lea esta guía para identificar la competencia que se trabajará.
2. Revise las sesiones en orden, desde modelado básico hasta decisiones con casos límite.
3. Compare los ejemplos y observe cómo cambia la solución cuando cambian los datos.
4. Resuelva los ejercicios verificando siempre entradas, procesos, salidas y pruebas.
5. Cierre el capítulo con la evaluación como evidencia de desempeño.

## Preguntas guía

- ¿Qué hace que una solución sea correcta y no solo funcional?
- ¿Cómo cambia un algoritmo cuando aparecen condiciones o casos especiales?
- ¿Qué errores aparecen cuando no se distinguen bien los tipos de datos?

## Evidencia esperada

Al terminar este capítulo, el estudiante debe poder resolver problemas básicos con secuencia y decisiones, justificando la lógica usada y comprobando su solución con casos de prueba simples.
