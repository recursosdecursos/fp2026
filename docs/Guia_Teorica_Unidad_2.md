# Capítulo 2. Guía teórica de resolución de problemas iterativos y procesamiento de datos

Este capítulo desarrolla la competencia de resolver problemas que requieren repetición, recorrido de datos y organización de información. El foco no está en memorizar estructuras, sino en elegir la estrategia adecuada para procesar conjuntos de datos de forma clara y verificable.

## Competencias del capítulo

- Identificar cuándo un problema requiere repetición definida o condicionada.
- Procesar colecciones de datos con criterios de búsqueda, acumulación y conteo.
- Organizar información en listas, matrices y diccionarios según la necesidad del problema.
- Analizar resultados parciales para detectar errores lógicos en procesos iterativos.

## Idea central

Cuando los problemas crecen, ya no basta una secuencia lineal. Se necesita recorrer, comparar, acumular, buscar y organizar datos para construir soluciones escalables.

## Ruta de lectura del capítulo

1. Lea esta guía para ubicar la competencia del capítulo.
2. Revise primero los patrones de repetición y luego las estructuras de datos básicas.
3. Observe en cada sesión cómo se conserva o transforma la información.
4. Compare soluciones distintas para el mismo problema según la estructura elegida.
5. Use la evaluación para demostrar dominio del procesamiento iterativo.

## Preguntas guía

- ¿Cuándo conviene usar `for` y cuándo `while`?
- ¿Qué información conviene guardar y cuál solo procesar?
- ¿Cómo influye la estructura de datos en la claridad de la solución?

## Evidencia esperada

Al terminar este capítulo, el estudiante debe poder diseñar soluciones iterativas para procesar colecciones de datos, justificando la estructura y el recorrido elegidos.
